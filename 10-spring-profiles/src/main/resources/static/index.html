üöÄ How to Activate Profiles

Method 1: Maven Command

mvn spring-boot:run -Dspring-boot.run.profiles=dev

Method 2: Environment Variable

export SPRING_PROFILES_ACTIVE=dev
mvn spring-boot:run

Method 3: Application Properties

# In application.properties
spring.profiles.active=dev

Method 4: IDE (IntelliJ IDEA)

Run Configuration ‚Üí Environment Variables ‚Üí SPRING_PROFILES_ACTIVE=dev

üìÅ Configuration Files

File Purpose Database
application.properties Default configuration H2 (default)
application-dev.properties Development environment H2 (dev)
application-prod.properties Production environment PostgreSQL
application-test.properties Test environment H2 (test)

üé≠ Profile-Specific Beans

@Configuration
@Profile("dev")
public class DevDatabaseConfig {
    @Bean
    public DatabaseInfo databaseInfo() {
        return new DatabaseInfo("H2 In-Memory", "dev");
    }
}

@Configuration
@Profile("prod")
public class ProdDatabaseConfig {
    @Bean
    public DatabaseInfo databaseInfo() {
        return new DatabaseInfo("PostgreSQL", "prod");
    }
}

üîß Profile Groups (NEW in Spring Boot)

Activate multiple profiles together:

# In application.properties
spring.profiles.group.production=prod,monitoring
spring.profiles.group.development=dev,debug

# Activate with:
spring.profiles.active=production  # Activates both prod and monitoring

üß™ How to Test

  1. Run with dev profile: mvn spring-boot:run -Dspring-boot.run.profiles=dev
  2. Check config: GET /api/config
  3. View profiles: GET /api/config/profiles
  4. Check actuator: GET /actuator/env
  5. Try different profiles: prod, test, default
  6. Compare configurations between profiles

üìñ Key Concepts Demonstrated

  • @Profile: Conditional bean registration based on active profiles
  • Property Files: application-{profile}.properties pattern
  • @ConditionalOnProperty: Conditional beans based on properties
  • Environment Object: Programmatic profile checking
  • Profile Groups: Activate multiple profiles together
  • Default Profile: Fallback when no profile is active
  • @ConfigurationProperties: Type-safe configuration
  • Actuator Integration: View environment via /actuator/env
