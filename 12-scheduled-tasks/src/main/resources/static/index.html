üîå REST API Endpoints

Base URL: http://localhost:8092

GET /api/scheduler/status

View execution counts of all scheduled tasks

curl http://localhost:8092/api/scheduler/status
POST /api/scheduler/dynamic/fixed-delay

Schedule a dynamic task with fixed delay

curl -X POST "http://localhost:8092/api/scheduler/dynamic/fixed-delay?taskName=MyTask&delaySeconds=10"
POST /api/scheduler/dynamic/cron

Schedule a dynamic task with cron expression

curl -X POST "http://localhost:8092/api/scheduler/dynamic/cron?taskName=HourlyTask&cronExpression=0+0+*+*+*+*"
POST /api/scheduler/dynamic/one-time

Schedule a one-time task

curl -X POST "http://localhost:8092/api/scheduler/dynamic/one-time?taskName=DelayedTask&delaySeconds=30"
GET /api/scheduler/dynamic

List all dynamic tasks

curl http://localhost:8092/api/scheduler/dynamic
DELETE /api/scheduler/dynamic/{taskName}

Cancel a dynamic task

curl -X DELETE http://localhost:8092/api/scheduler/dynamic/MyTask

üìÖ Cron Expression Guide

Cron format: second minute hour day month weekday

Expression Meaning Use Case
0 * * * * * Every minute Status checks
0 0 * * * * Every hour Hourly reports
0 0 9 * * MON-FRI Weekdays at 9 AM Daily reports
0 0/15 * * * * Every 15 minutes Monitoring
0 0 0 * * * Daily at midnight Cleanup tasks
0 0 12 1 * * 1st of month at noon Monthly reports

‚öôÔ∏è Scheduling Types Explained

1. Fixed Rate

@Scheduled(fixedRate = 5000, initialDelay = 2000)
public void fixedRateTask() {
    // Runs every 5 seconds, regardless of previous execution
}

‚úÖ Use when: Task execution time is predictable and short

2. Fixed Delay

@Scheduled(fixedDelay = 3000, initialDelay = 1000)
public void fixedDelayTask() {
    // Waits 3 seconds after previous execution completes
}

‚úÖ Use when: You want guaranteed gap between executions

3. Cron Expression

@Scheduled(cron = "0 0 * * * *")  // Every hour
public void cronTask() {
    // Runs at specific times
}

‚úÖ Use when: Need complex schedules (daily at 9 AM, weekdays only, etc.)

üîß Conditional Scheduling

@Scheduled(cron = "0 0 2 * * *")  // Daily at 2 AM
@ConditionalOnProperty(
    name = "app.scheduling.cleanup-enabled", 
    havingValue = "true"
)
public void dailyCleanupTask() {
    // Only runs when property is enabled
}

Enable in application.properties:

app.scheduling.cleanup-enabled=true

üìñ Key Concepts Demonstrated

  • @Scheduled: Declarative task scheduling
  • @EnableScheduling: Enable scheduling support
  • Fixed Rate: Execute at regular intervals
  • Fixed Delay: Wait between executions
  • Cron Expressions: Complex time-based schedules
  • TaskScheduler: Programmatic scheduling
  • @ConditionalOnProperty: Conditional scheduling
  • ScheduledFuture: Control task lifecycle
